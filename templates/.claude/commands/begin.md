# Begin - Intelligent Task Router

**Purpose:** Understand your task, analyze context, and recommend the optimal workflow.

**TAC Principle:** This command embodies all 5 TAC principles by guiding you through the right process for your specific task.

---

## Instructions

You are the intelligent entry point for this project. Your job is to:

1. **Setup project if needed** - Generate project-context.md if missing
2. **Understand the task** - What does the user want to accomplish?
3. **Analyze context** - What's the current state of the project?
4. **Recommend workflow** - Which commands or ADW workflow is best?
5. **Explain reasoning** - Why this approach fits their needs
6. **Provide next steps** - Exact commands to execute

## Step 0: First-Time Setup Check

**IMPORTANT:** Before doing anything else, check if `.claude/project-context.md` exists.

### If `.claude/project-context.md` DOES NOT EXIST:

Switch to **Setup Mode** - you will interview the user and generate `project-context.md` for them.

**Interview Process:**

1. **Detect project type automatically:**
   - Check for `package.json` → Node.js/JavaScript/TypeScript
   - Check for `pyproject.toml`, `requirements.txt`, `setup.py` → Python
   - Check for `go.mod` → Go
   - Check for `Cargo.toml` → Rust
   - Check for `pom.xml`, `build.gradle` → Java
   - Check files in root directory to infer framework

2. **Read existing files to minimize questions:**
   - If `README.md` exists → extract project name, purpose, description
   - If `package.json` exists → extract dependencies, scripts
   - If `pyproject.toml` exists → extract dependencies, Python version
   - Scan directory structure to understand layout

3. **Ask only what you cannot infer:**

   Start with: "I notice this is a [detected language/framework] project. I'll help you set up the agentic coding framework by creating a project-context.md file. I've analyzed your codebase and detected the following:"

   **Show what you detected:**
   ```
   - Language/Framework: [X]
   - Key Dependencies: [list]
   - Directory Structure: [overview]
   - Existing Scripts: [list from package.json/etc]
   ```

   **Then ask only missing information:**

   a) **Project Details** (if not in README):
      - "What's the primary purpose of this project in 1-2 sentences?"
      - "What stage is it in? (MVP / Development / Production)"

   b) **Development Workflow** (if scripts don't reveal):
      - "What command runs your dev server?" (if not obvious)
      - "What command runs your tests?" (if not obvious)
      - "What command builds for production?" (if not obvious)

   c) **Environment Variables** (critical):
      - Check for `.env.example`, `.env.template` → list vars found
      - Ask: "Are there any environment variables not in .env.example that I should know about?"

   d) **Coding Conventions** (only if complex):
      - "Any specific naming conventions or code organization rules I should follow?"
      - If they say "standard" or "none" → use language defaults

   e) **Critical Context** (always ask):
      - "What should AI NEVER do in this project?"
      - "What should AI ASK before doing?"

4. **Generate `.claude/project-context.md`:**

   Use the Write tool to create the file with all gathered information. Use this structure:

   ```markdown
   # [Project Name] - Project Context

   **Purpose:** [Extracted or provided purpose]

   **Stage:** [Provided stage]

   **Repository:** [From git remote if available]

   ---

   ## Tech Stack

   ### Core Technologies
   - **Language:** [Detected language + version]
   - **Framework:** [Detected framework + version]
   - **Database:** [Detected from dependencies or "Not applicable"]

   ### Key Dependencies
   [List from package.json/pyproject.toml/etc with versions]

   ### Infrastructure
   - **Hosting:** [If mentioned, else "To be determined"]
   - **CI/CD:** [Check for .github/workflows or .gitlab-ci.yml]

   ---

   ## Codebase Structure

   [Auto-generated based on directory scan]

   ---

   ## Environment Variables

   [List from .env.example with descriptions]

   ---

   ## Coding Conventions

   [Defaults for the language + any user-specified rules]

   ---

   ## Development Workflow

   ### Initial Setup
   ```bash
   [Detected install command]
   ```

   ### Daily Development
   ```bash
   # Start dev server
   [Detected dev command]

   # Run tests
   [Detected test command]

   # Build
   [Detected build command]
   ```

   ---

   ## What AI Should Know

   ### Always Do
   - Run tests before committing
   - Follow the naming conventions above
   - [User-specified rules]

   ### Never Do
   - [User-specified forbidden actions]

   ### Ask First
   - [User-specified approval-required actions]

   ---

   **Last Updated:** [Current date]
   **Auto-generated by:** /begin command
   ```

5. **Confirm with user:**

   After generating, display the file content and say:

   ```
   ✅ I've created `.claude/project-context.md` for you!

   📄 Here's what I generated:
   [Show summary of key sections]

   You can review the full file at: .claude/project-context.md

   Would you like me to:
   1. Proceed with your original task?
   2. Update anything in project-context.md first?
   ```

6. **Save time estimate:**
   - Setup interview: 3-5 minutes (AI asks smart questions)
   - Manual template: 15-20 minutes (user fills template)
   - **Time saved: 10-15 minutes** ✨

### If `.claude/project-context.md` EXISTS:

Proceed to **Task Routing Mode** (continue to Step 1 below).

## Step 1: Gather Context

Read the following in order:

1. **Project Context:**
   - Read `.claude/project-context.md` to understand the tech stack, conventions, and project state

2. **Current State:**
   - Run `git status` to see current branch and changes
   - Run `git branch` to see available branches
   - Check if there are uncommitted changes

3. **User Intent:**
   - Ask the user: "What would you like to work on today?"
   - Listen for keywords: "feature", "bug", "fix", "refactor", "test", "deploy", "understand", "setup"

## Step 2: Analyze Task Type

Based on the user's response, classify their intent:

### New Development Work
- **Keywords:** "add", "create", "build", "implement", "new feature"
- **Indicators:** No GitHub issue mentioned, exploratory
- **Suggest:** Manual workflow with slash commands

### GitHub Issue Work
- **Keywords:** "issue #123", "fix issue", "work on #45"
- **Indicators:** Specific issue number mentioned
- **Suggest:** ADW automated workflow

### Bug Fix
- **Keywords:** "bug", "broken", "not working", "error", "fix"
- **Indicators:** Something isn't functioning correctly
- **Suggest:** `/bug` planning or ADW if issue exists

### Understanding Codebase
- **Keywords:** "understand", "explain", "how does", "where is"
- **Indicators:** Learning or investigation
- **Suggest:** `/prime` or codebase exploration

### Maintenance/Refactoring
- **Keywords:** "refactor", "cleanup", "update dependencies", "chore"
- **Indicators:** Code quality improvement
- **Suggest:** `/chore` planning

### Testing/Validation
- **Keywords:** "test", "validate", "check if working"
- **Indicators:** Verification needed
- **Suggest:** `/test` or `/review`

### Documentation
- **Keywords:** "document", "write docs", "explain how"
- **Indicators:** Documentation needed
- **Suggest:** `/document`

## Step 3: Check Prerequisites

Before recommending a workflow, verify:

1. **For ADW workflows:**
   - ✓ Environment variables set? (`GITHUB_REPO_URL`, `ANTHROPIC_API_KEY`)
   - ✓ `adw/` directory exists?
   - ✓ GitHub issue exists?
   - If missing → Recommend manual workflow

2. **For Manual workflows:**
   - ✓ `.claude/project-context.md` exists?
   - ✓ Commands available in `.claude/commands/`?
   - If missing → Recommend setup first

3. **Knowledge Gaps:**
   - Is this using new technology the AI might not know?
   - Is this a complex domain requiring documentation?
   - If yes → Recommend `/knowledge_gap_check` first

- Show `git log` recent commits
- Check for GitHub issues with `gh issue list`
- Suggest reviewing project goals in project-context.md

- Ask for symptoms
- Check test failures: `/test`
- Recommend `/bug` workflow with investigation phase

### Case 3: "I want to improve code quality"
- Suggest `/chore` for refactoring
- Check test coverage
- Review validation checklists

- `/begin` will automatically detect missing project-context.md
- Switch to Setup Mode (Step 0)
- Interview user and auto-generate project-context.md
- Then proceed with their task
