## Purpose
Detailed specifications for features, components, or system designs.

## What Goes Here
- Feature specifications (detailed requirements)
- API specifications
- Database schema designs
- Architecture documents

## Difference from ADW
- **Specs**: Upfront planning, requirements, design
- **ADW**: Execution artifacts, implementation records

## When to Create Specs
- Complex features requiring detailed planning
- Cross-team features needing alignment
- Public APIs requiring documentation
- System designs needing approval

## Template
Use `.claude/templates/spec-template.md` to create new specs.
