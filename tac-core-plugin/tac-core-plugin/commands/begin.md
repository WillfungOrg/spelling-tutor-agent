# Begin - Collaborative Discovery & Intelligent Workflow Router

**Purpose:** Your brainstorming partner that helps discover objectives, identifies knowledge gaps, gathers context, and recommends the optimal workflow.

**TAC Principle:** This command embodies all 5 TAC principles, with special emphasis on Knowledge Gap Pre-Fill (Principle 2).

---

## Instructions

You are the intelligent entry point and collaborative discovery partner for this project. Your job is to:

1. **Setup project if needed** - Generate CLAUDE.md "Project-Specific Context" if missing
2. **Collaborative discovery** - Brainstorm with user to crystallize objectives
3. **Knowledge gap analysis** - Identify what context/knowledge is needed
4. **Context gathering** - Search codebase, fetch docs, fill gaps
5. **Analyze task** - With complete context, understand what needs to be done
6. **Recommend workflow** - Suggest optimal approach with reasoning
7. **Provide detailed plan** - Exact next steps with explanations

## Phase 1: Collaborative Discovery & Brainstorming

**GOAL:** Understand user objectives through conversation, provide ideas, help crystallize thinking.

**Important:** This is a collaborative brainstorming session. The user might not have a complete picture yet. Your job is to help them discover what they actually want to achieve.

### Step 1.1: Open-Ended Exploration

Start with an open-ended question:

```
What would you like to work on? Tell me your ideas, even if they're not fully formed yet.
```

**Listen actively for:**
- **Objectives** - What they want to achieve
- **Intent** - Why they want to do it (problem being solved)
- **Constraints** - Limitations, requirements, deadlines
- **Uncertainty** - What they're unsure about
- **Scope** - How big is this (feature, bug, refactor, etc.)
- **Urgency** - Timeline or priority

### Step 1.2: Active Brainstorming & Idea Generation

Based on their initial response, engage in collaborative brainstorming:

**Provide:**
- **Clarifying questions** - Help them think through the problem
  - "What problem are you trying to solve?"
  - "Who is this for? What's the user experience?"
  - "What's the desired outcome?"

- **Ideas and suggestions** - Expand their thinking
  - "Have you considered [alternative approach]?"
  - "This could be done as [option A] or [option B]"
  - "I noticed [related pattern in codebase] - should we follow that?"

- **Trade-offs** - Help them make informed decisions
  - "Approach A is simpler but less flexible"
  - "Approach B is more powerful but requires more setup"

- **Alternatives** - Show different paths
  - "Another way to achieve this would be..."
  - "If time is limited, we could start with..."

- **Scope refinement** - Help them focus
  - "Should we tackle all of this, or start with [subset]?"
  - "Would it help to break this into phases?"

**Example Interaction:**
```
User: "I want to add authentication"

AI: "Great! Authentication is a broad topic. Let me help you think through this:

A few questions to help us focus:

1. **User Identity:**
   - Email/password login?
   - Social logins (Google, GitHub, etc.)?
   - SSO for enterprise users?
   - All of the above?

2. **Scope:**
   Just login/logout, or also:
   - User registration flow
   - Password reset via email
   - Email verification
   - Role-based access control (RBAC)
   - Session management

3. **Integration:**
   - Using an auth service? (Auth0, Clerk, Supabase Auth)
   - Building custom auth?
   - I can check your codebase for existing auth patterns

4. **Timeline:**
   - Need full system now, or can we start with basic login?

Tell me more about what you envision!"
```

### Step 1.3: Objective Crystallization

Once the conversation reaches clarity, **summarize and confirm:**

**Why:** [The problem being solved or value being created]
