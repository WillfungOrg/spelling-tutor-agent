# Meta-ADW Custom ADW Designer

**Purpose:** Design custom ADW workflows for novel/unique tasks that don't match existing templates.

**When to use:** When decision engine confidence < 0.5 (no good template match) or user explicitly requests custom workflow.

---

## Your Task

You are a Meta-ADW system architect. Design a custom ADW workflow for a novel task that doesn't fit existing templates. Create both the workflow script and corresponding slash command template.

## Input Format

You will receive a TaskAnalysis JSON:

```json
{
  "task_type": "...",
  "complexity_score": <float>,

```

## Design Process

### 1. Analyze Requirements

Extract unique characteristics:
- What makes this task novel? (New tools, new patterns, new integrations)
- What existing ADW phases apply? (plan, build, test, review, document)
- What custom phases are needed? (setup, deploy, monitor, etc.)
- What validation is critical? (L5/L6/L7 patterns)

### 2. Design Workflow Structure

Determine workflow phases:

**Standard Phases (reuse from existing ADWs):**
- `plan` - Create implementation plan
- `build` - Execute implementation
- `test` - Run tests with auto-fix
- `review` - Code review with spec compliance
- `document` - Generate documentation

**Custom Phases (design from scratch):**
- `setup` - Environment/infrastructure setup
- `migrate` - Database or data migration
- `deploy` - Deployment orchestration
- `monitor` - Post-deployment monitoring
- `rollback` - Rollback capability
- `validate` - Specialized validation
- `benchmark` - Performance testing

### 3. Design Validation Strategy

Map validation_needs to TAC patterns:

- **basic** → Unit tests + integration tests
- **thorough** → L5 closed-loop (agent verifies in real environment)
- **critical** → L6 specialized agents (security, performance, accessibility)
- **l5_closed_loop** → Agent has access to logs, verifies deployment
- **l6_specialized** → Multiple validation agents for different aspects
- **l7_zero_touch** → Full automation with confidence metrics

### 6. Update Template Index

Add new template to `.tac/template_index.json`:

```json
{
  "templates": {
    "custom_<name>.md": {
      "t": <token_estimate>,
      "g": ["custom", "learned"],
      "s": "<50-char summary>"
    }
  }
}
```
