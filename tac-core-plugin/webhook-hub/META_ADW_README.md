# Meta-ADW Strategy System

**Intelligent TAC strategy advisor that recommends optimal workflows for any task.**

---

## What is Meta-ADW?

Meta-ADW is a decision-making layer above existing ADW workflows. It:
- Interviews users about their tasks
- Analyzes complexity and requirements
- Recommends optimal workflow/template combinations
- Validates plans for completeness
- Learns from execution outcomes over time

**Problem it solves:** Decision paralysis from 30+ available templates. No clear guidance on validation strategy (L5 vs L6 vs L7).

---

## Quick Start

```bash
# Run Meta-ADW on a GitHub issue
cd adw/
uv run meta_strategy.py <issue-number>

# Example
uv run meta_strategy.py 123

# With heavy model set (for complex tasks)
uv run meta_strategy.py 123 --model-set heavy
```

## How It Works

### 1. Interview Phase
Asks 7 key questions:
- Task type (feature/bug/chore/etc.)
- Complexity level (simple/moderate/complex)
- User interaction (UI/backend/full-stack)
- Testing requirements (unit/integration/e2e)
- Validation needs (basic/thorough/critical)
- Timeline (quick-patch/normal/comprehensive)
- Novelty (standard/novel/unique)

### 2. Analysis Phase
Analyzes task characteristics:
- Complexity scoring (1-10)
- Estimated effort (hours)
- Risk factors
- Required validation depth (TAC L5/L6/L7)
- Recommended model set (base/heavy)

### 3. Recommendation Phase
Uses decision engine with pattern matching:
- **Keyword matching** (40%): Task requirements vs template use cases
- **Complexity matching** (30%): Task complexity vs template complexity
- **Validation depth** (30%): Required validation vs template validation level
- **Historical success**: Boosts based on past performance

Returns top 3 recommendations with confidence scores.

### 4. Execution Phase
- Displays recommendations with reasoning
- User confirms or overrides selection
- Executes chosen workflow
- Logs execution for learning system

### 5. Learning Phase
- Collects feedback (execution time, success/failure)
- Detects "false fixes" (agent says done but issue persists)
- Updates pattern database with task→strategy mappings
- Retrains decision weights after 10+ executions

## Direct Template Usage

You can use strategy templates independently:

```bash
# Interview only
claude --command /strategy/interview "Add OAuth authentication"

# Analyze task (requires interview JSON output)
claude --command /strategy/analyze '<interview-json>'

# Recommend strategy (requires analysis JSON)
claude --command /strategy/recommend '<analysis-json>'

# Validate a plan
claude --command /strategy/validate_plan '<plan-content>'

# Design custom ADW for novel tasks
claude --command /strategy/design_custom_adw '<task-analysis-json>'
```

## Configuration

### Template Index
`.tac/template_index.json` - Lightweight catalog (~5KB) of all templates
- Generated by `scripts/build_template_index.py`
- Rebuild: `python scripts/build_template_index.py --rebuild`

### Learning Data
`.tac/learning/pattern_database.json` - Learned task→strategy patterns
`.tac/learning/decision_weights.json` - Pattern matching weights
`.tac/learning/execution_logs/` - Execution history logs

### Decision Weights
Default weights (can be auto-adjusted by learning system):
```json
{
  "template_match": 0.4,
  "complexity_match": 0.3,
  "validation_depth": 0.3
}
```

## Validation Strategies

Meta-ADW maps validation needs to TAC patterns:

- **basic** → Unit + integration tests
- **thorough** → TAC L5 (closed-loop with real environment)
- **critical** → TAC L6 (specialized validation agents)
- **l5_closed_loop** → Agent verifies in production-like environment
- **l6_specialized** → Multiple validation agents (security, performance, accessibility)
- **l7_zero_touch** → Full automation with confidence metrics

---

## When to Use Meta-ADW vs Direct ADW

**Use Meta-ADW when:**
- Unsure which workflow to use
- Task is novel or unique
- Want validation strategy recommendations
- Building learning data for future tasks

**Use direct ADW when:**
- You know exactly which workflow you need
- Simple/standard task (e.g., clear bug fix)
- Speed is critical

## Learning System

### How It Learns
1. **Collects feedback** after each execution:
   - Execution time
   - Issue resolution status
   - Test pass rates
   - PR review feedback

2. **Detects false fixes**:
   - Checks if issue reopened within 7 days
   - Monitors for production errors

3. **Updates patterns**:
   - Stores successful task→strategy mappings
   - Tracks failure patterns
   - Updates template success rates

4. **Retrains weights** (after 10+ executions):
   - Analyzes success correlations
   - Adjusts weights to prioritize what works
   - Improves recommendations automatically

### Success Metrics Tracked
- Template selection accuracy (user acceptance rate)
- Average token usage per execution
- "False fix" rate over time
- Time to decision (interview → recommendation)
- Custom ADW creation frequency

## Troubleshooting

### Template Index Missing
```
Error: Template index not found at .tac/template_index.json
Solution: python scripts/build_template_index.py --rebuild
```

### Low Confidence Scores
- Indicates novel task with no good template match
- Meta-ADW will suggest custom ADW creation
- Or manually select closest alternative

### Learning Data Corrupted
- Delete `.tac/learning/pattern_database.json`
- System will reinitialize with defaults
- Previous learning will be lost

## Advanced Usage

### Custom ADW Creation
For novel tasks with confidence < 0.5:
```bash
# Meta-ADW triggers automatically, or manually:
claude --command /strategy/design_custom_adw '<task-analysis-json>'
```

Generates:
- `adw/learned/adw_custom_<name>.py` - Workflow script
- `.claude/commands/learned/<name>.md` - Slash command template
- Updates template index

### Integration with Existing ADWs
Add `--use-meta-adw` flag to existing workflows:
```bash
# Future enhancement (not yet implemented)
uv run adw_plan_iso.py 123 --use-meta-adw
```

## Limitations

- First execution has no learning data (recommendations based on keywords only)
- Custom ADW designer requires heavy model (Opus recommended)
- Token usage increases with task complexity
- Learning requires 10+ executions before patterns emerge
- Concurrent executions may conflict on pattern_database.json
